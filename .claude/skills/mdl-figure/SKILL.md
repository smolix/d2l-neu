# mdl-figure — generate an illustrative figure in the d2l house style

Illustrative figures in this book are **pre-generated static SVGs**, never drawn
inline. A committed matplotlib generator (`tools/gen_mdl_figures.py`) owns one
shared house style; notebooks and chapter prose reference the output files with
**no drawing code** — exactly like the slide SVGs. This skill is how you add or
change one of those figures correctly.

## When to use / not use

- **Use** for conceptual/schematic/geometric pictures: a vector and its
  projection, an angle, a hyperplane, the action of a matrix, an eigen-ellipse,
  the SVD's rotate–scale–rotate, a flow/process diagram, etc.
- **Do NOT use** for a plot that *teaches a computed result* the prose is
  discussing (a loss curve, a fitted line, a sampled histogram). Those stay as a
  short `d2l.plot(...)` cell **in the notebook** — they are teaching code, not
  decoration. (See CLAUDE.md → "Content authoring": *code teaches; it does not
  draw*.)
- **Never** put figure-drawing matplotlib plumbing in a notebook cell.

## The generator (source of truth for style)

`tools/gen_mdl_figures.py` defines, once:

- a fixed palette — `BLUE ORANGE GREEN GRAY LIGHT` (use these by name, never raw
  hex per-figure);
- `plt.rcParams` with `svg.hashsalt="mdl-la"`, `svg.fonttype="path"`,
  `mathtext.fontset="cm"` — **do not remove these**; the hashsalt + the
  `metadata={'Date': None}` in `save()` make re-runs **byte-for-byte identical**
  so figures diff cleanly in git;
- shared helpers — `arrow()`, `vlabel()`, `right_angle()`, `clean_axes()`,
  `axis_cross()` — reuse them so every figure reads the same;
- `save(fig, name)` — writes `img/<name>.svg` with a tight bbox and records it
  for the end-of-run verification.

Read the top ~150 lines of the generator before adding a figure; mirror an
existing `fig_*` that is closest to what you need.

## Steps to add a figure

1. **Pick an id + prefix.** Names are `mdl-<chapterslug>-<id>` (e.g. Linear
   Algebra uses `mdl-la-…`: `mdl-la-projection`). Reuse the chapter's existing
   prefix; for a new chapter, choose a short slug and use it consistently. IDs
   never change once written (they're referenced by `:label:`).

2. **Write `fig_<id>()`** in `tools/gen_mdl_figures.py`, near the other figures
   of that chapter. Build the axes with `clean_axes()`/`axis_cross()`, draw with
   `arrow()`/`right_angle()`/`vlabel()` and the palette, and **compute real
   numbers** where the picture must be exact (apply the matrix, call
   `np.linalg.eig`/`svd`, sample) — the figures teach by being accurate, not
   sketched. End with `save(fig, "mdl-<prefix>-<id>")`.

3. **Register it** by adding `fig_<id>` to the `FIGURES` list (under the
   chapter's comment block).

4. **Generate** (CPU, fast):

   ```bash
   .venv-pytorch/bin/python tools/gen_mdl_figures.py
   ```

   It writes every `img/mdl-*.svg` and self-verifies each is present, non-empty,
   and valid SVG. Confirm your new file changed and **re-run once more** to
   confirm `git diff` is now empty (idempotence).

5. **Include it in the chapter `.md`** (the source of truth — never the `.qmd`):

   ```markdown
   ![A vector $\mathbf{v}$ and its projection onto $\mathbf{u}$.](../img/mdl-la-projection.svg)
   :label:`fig_mdl-la-projection`
   ```

   Reference it from prose with `` :numref:`fig_mdl-la-projection` ``.

6. **Render & verify**, then commit the `.svg` **and** the generator change
   together:

   ```bash
   make html
   # confirm 0 unresolved refs and the figure renders; preview at :8000
   ```

## Gotchas (these silently break the label/caption)

- **No `]` in a caption** — a `]` truncates the markdown alt-text, so the
  `:label:` lands on its own and the `:numref:` fails to resolve. Write matrices
  in captions as `\left(\begin{smallmatrix}2&1\\1&2\end{smallmatrix}\right)`,
  never `[[2,1],[1,2]]`.
- **One style per chapter.** Don't mix hand-drawn SVGs, inline matplotlib, and
  generated figures in the same chapter — reproduce any straggler in this
  generator's style.
- **Idempotence is load-bearing.** If a re-run produces a non-empty `git diff`,
  something added a timestamp/random id — check you used `save()` (not a bare
  `fig.savefig`) and didn't drop `svg.hashsalt`/`metadata={'Date': None}`.
