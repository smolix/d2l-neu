# D2L-Neu Project Context

## What this is
Rebuild of the "Dive into Deep Learning" (d2l.ai) book using Quarto, replacing
the custom d2l-book/Sphinx pipeline. Source content comes from d2l-en (the
original d2l repository at github.com/d2l-ai/d2l-en).

## Repository structure
```
tools/                      Build tools (Python scripts)
  d2l_preprocess.py         Converts d2l-en .md → Quarto .qmd (multi-framework tabs)
  fix_crossref_numbers.py   Post-render HTML fix for chapter/section/figure numbering
  gen_pdf.py                Single-framework .qmd for PDF rendering
  fix_latex.py              Post-processes LaTeX for hierarchical chapter/section structure
  gen_notebooks.py          Per-framework Jupyter notebooks (parallelized)
  gen_slides.py             Per-framework Reveal.js slides (parallelized)
  build_lib.py              Extracts #@save code into d2l Python package
build.sh                    Unified build script (html|pdf|notebooks|slides|lib|all|clean)
_quarto.yml                 Quarto book project config
_d2l-theme.scss             Custom SCSS theme (d2l.ai blue palette)
_d2l-tabs.html              Tab sync JS + sidebar collapse JS
_d2l-style.css              Additional CSS overrides
_svg-to-pdf.lua             Lua filter for SVG→PDF in LaTeX
chapter_*/                  Quarto .qmd chapter files (generated from d2l-en)
img/                        Images (copied from d2l-en, no PDFs — SVG→PDF at build time)
d2l/                        Generated d2l Python package (torch/tf/jax/mxnet)
d2l.bib                     Bibliography (487 entries, audited, 91% with DOIs/URLs)
static/                     Fonts (Source Serif 4, Source Sans 3, Inconsolata) + LaTeX preamble
```

## Build pipeline
```bash
./build.sh html        # Preprocess → Quarto render → fix numbering (post-render)
./build.sh pdf pytorch # Generate single-fw .qmd → Quarto render → fix LaTeX → XeLaTeX ×2
./build.sh notebooks   # Generate per-framework .qmd → quarto convert (10 threads)
./build.sh slides      # Generate per-framework .qmd → quarto render revealjs (10 threads)
./build.sh lib         # Extract #@save blocks from d2l-en → d2l/*.py
./build.sh all         # All of the above
./build.sh clean       # Remove _book/ _pdf/ _notebooks/ _slides/
```

## Key design decisions
- Source .qmd files are GENERATED by d2l_preprocess.py from d2l-en .md files.
  Do NOT manually edit .qmd files — edits will be overwritten on next preprocess run.
  All fixes must go into the preprocessor or the d2l-en source.
- HTML uses multi-framework tabs (PyTorch/TF/JAX/MXNet) with synchronized selection.
  PDF/notebooks/slides are single-framework.
- Chapter numbering: Quarto assigns file-position numbers; fix_crossref_numbers.py
  remaps them to logical chapter numbers (e.g., file 54 → chapter 8.1).
- Equations: ALL display equations are auto-numbered by the preprocessor
  ({#eq-SECTION-auto-N} labels).
- Tab sync works on file:// URLs via custom inline JS (not Quarto's ES module).
- Sidebar: part headers are clickable (JS), first child hidden (CSS), sections
  default collapsed with localStorage persistence.
