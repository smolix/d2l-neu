## Cross-cutting observations (apply across decks)

1. **The one real depth gap is a *computed* numerical-failure demo.** The
   overflow/underflow story is the chapter's signature numerical lesson and it is
   currently **asserted in prose** ("`exp(1000)=∞`, the ratio is `∞/∞=NaN`") and
   set as *exercise 3* of the concise section, but **no executed cell ever shows
   it on a slide.** The calculus deck is the gold standard here: its
   "Verifying numerically" slide *runs* the difference quotient and *shows* the
   floating-point cancellation. §4 should do the same once: a tiny cell that
   computes `softmax(tensor([1000., 0., 0.]))` → `[nan, 0., 0.]` and, beside it,
   the framework's `F.cross_entropy(logits, y)` returning a finite number, plus
   the benign-logits agreement check. This is the highest-value single change in
   the whole chapter (rubric #4 *and* #6). It needs **one new tagged code cell**
   in the scratch *or* concise source (author task; flagged here, not done) and a
   re-run/capture; the slide then references it by `@id`. Until that cell exists,
   the slide can only assert, which is exactly the "code with no computed result
   on the slide" pattern we are fixing. See scratch (preferred home) and concise
   entries.

2. **Per-framework `only=`/`except=` scoping is already done where it matters,
   and done well.** The chapter's genuine framing divergences are correctly
   scoped: the concise deck has four `only="<fw>"` "Hand the loss the logits"
   slides (PyTorch `F.cross_entropy` vs TF `from_logits=True` vs Optax
   `softmax_cross_entropy_with_integer_labels` vs MXNet `SoftmaxCrossEntropyLoss`);
   classification has `only="jax"` (stateless/functional validation step) and
   `only="mxnet"` (the `get_scratch_params` fallback); image-classification has
   channel-first (`only="pytorch,mxnet"`) vs channel-last (`only="tensorflow,jax"`)
   layout slides and an `only="mxnet"` minibatch-shape slide; scratch has
   `only="pytorch"` on the accuracy-sweep slide. **No additional scoping is
   needed.** One thing to *verify at build time*: the JAX `loss`/`accuracy`
   signatures differ (they take `params, X, y, state`), so confirm the shared
   `@id` code slides render the right per-fw variant in the JAX deck and don't
   show a confusing signature without its `only="jax"` context (the existing
   scoping appears to cover this, but it is the one place to eyeball across all
   four decks).

3. **The "two faces of the logits" through-line is the chapter's spine and is
   already threaded** via `mdl-clf-loss-accuracy.svg` appearing as a bookend in
   softmax-regression (§4.1 divider "Two Faces of the Scores"), classification

## Deck 1: `softmax-regression.md` (§4.1)  ·  18 slides

**Depth verdict: EXCELLENT (good).** This is the technical heart of the chapter
and the current deck already lands every must-hit anchor. No structural rework
needed; only micro-polish.

What it already nails (do not lose in any regeneration):

- scores → probabilities → loss stated as one pipeline (cover + motivation
  callout).
- one-hot as a vector in probability space; "why not just regress on the labels?"
  with the warn callout (`o_i ≥ 0`? `Σo_i = 1`?, no).
- softmax = exponentiate + normalize, with the Boltzmann/partition-function
  connection; **ranking preserved** (`argmax ŷ = argmax o`) → "predict by reading
  the biggest logit, softmax is for the *loss*."
- **translation invariance** (`softmax(o + c1) = softmax(o)`) → one redundant DOF
  → foreshadows the stability shift `c = −max o`.
- **two-class collapse to the sigmoid** `ŷ₁ = σ(o₁−o₂)`, staged with a fragment.
- MLE → sum of losses; cross-entropy `= −log ŷ_true`; the bound `≥ 0`.
- **the headline derivation**: `l = g(o) − yᵀo`, then
  `∂_{o_j} l = softmax(o)_j − y_j = ŷ_j − y_j`, framed as the exponential-family
  residual and tied back to linear regression. Staged with a fragment. This is
  the slide the whole deck is built around and it is done right.
- cross-entropy = code length (entropy / cross-entropy), the MLE-vs-coding duality.

**Core spine (already present):** (1) labels & the linear model → logits; (2) the
softmax (squish, rank-preserving, shift-invariant, sigmoid special case); (3)
cross-entropy as NLL and the `ŷ − y` gradient; (4) the same logits drive training
and evaluation.

**Must-show derivation (already staged, keep verbatim shape):**

    l(y, ŷ) = −Σ_j y_j log[ exp(o_j) / Σ_k exp(o_k) ]
            = log Σ_k exp(o_k) − Σ_j y_j o_j        (Σ_j y_j = 1)
            = g(o) − yᵀo,     g(o) = log Σ_k exp(o_k)
    ∂_{o_j} g(o) = softmax(o)_j      ⟹      ∂_{o_j} l = softmax(o)_j − y_j = ŷ_j − y_j

**Lead diagrams:** `img/softmaxreg.svg` (the single FC layer, used twice, cover
and "one affine function per class"); `img/mdl-clf-loss-accuracy.svg` (the §4
divider "Two Faces of the Scores," with its real worked numbers). Both reused;
**no new figure needed.**

**Code that earns its place:** none, this is a pure math/concept section with no
notebook (correctly: there is no `_notebooks/.../softmax-regression.ipynb`). Do
*not* add code; the calculus deck shows a math-only section can be top-tier.

**Cross-links / forward-points (already present, all good):** §3 linear regression
(same MLE recipe, same residual gradient); Boltzmann/stat-mech; information theory
(`sec_mdl-information_theory`) for entropy/CE/KL; temperature scaling as a
forward-point to modern calibration (the chapter prose has it, see refinement).

**Refinements (small, optional):**

- The chapter body has a genuinely modern, genuinely deep aside the deck drops:
  **calibration**, "a CE-trained net is *not* calibrated; modern nets are
  systematically overconfident (Guo et al. 2017); *temperature scaling* divides
  logits by a learned `T` and, being a uniform scale, preserves the argmax." This
  is a perfect rubric-#5 forward-point and ties back to the deck's own Boltzmann
  temperature. Consider **one** added slide (or a fragment on the "two faces"
  slide): "Probabilities, but not *calibrated*", `T`-scaling sharpens/softens
  confidences without moving the decision. High value, low risk; it modernizes

## Deck 6: `generalization-classification.md` (§4.6)  ·  30 slides

**Depth verdict: EXCELLENT (good).** A theory section done at the bar. It leads
with the motivating question ("memorizing is not learning; how much do you trust a
test score, and can we guarantee generalization *before* seeing the data?"),
stages the statistics cleanly, and **leads the VC content with the shattering
picture** exactly as the brief asks.

**Core spine (present):** (1) two errors, empirical (measurable) vs population
(wanted); for a *fixed* `f` the test miss-rate is an unbiased sample mean; (2) the
√n law (Bernoulli indicator → CLT → `±O(1/√n)`; the "2× precision = 4× data" tax;
variance capped at 0.25 → ≈10k for ±0.01 at 95 %; Hoeffding → ≈18.5k honest
finite-sample); (3) test-set *reuse* decays it (false discovery + adaptive
overfitting → hygiene rules); (4) learning theory: uniform convergence → VC
dimension (shattering) → the VC bound → why it is vacuous for deep nets.

**Must-show results (already staged with fragments, keep):**

    Empirical vs population error:
        ε_D(f) = (1/n) Σ 1(f(x_i) ≠ y_i)      vs      ε(f) = E_{(x,y)~P} 1(f(x) ≠ y)
    Square-root law:   ε_D(f) ≈ ε(f) ± O(1/√n)
